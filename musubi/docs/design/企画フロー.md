# 企画フロー

## 1. 言語化

以下を明確に定義し、AIの厳格なレビューを通す。マネタイズはいったん無視（バズってないのに設計しても無意味なので）。

- 価値
- ペルソナ
- 差別化点（技術ではなく価値の）

## 2. 機能定義

- 要件定義作成
- Vibe Design用にAIで情報を過不足なく整理

## 3. グランドデザイン作成

- まず要件定義をもとにVibe Coding
  - 骨子はFigmaより早い。人手でやるとしんどいので、叩き作らせてこっちで調整する。
  - 遷移案固まったらhtmlに吐き出してmakeに送る

- Figma MakeでVibe Design
  - 要件定義を渡して対話的にデザインしていく
- 参考サイトがある場合はh2dで
- レスポンシブ対応は不要（Vibe Codingで対応）
  - 繰り返しコンテンツのfigmaレイヤー名は統一
- AI感のでないコツは？

## 4. CM作成

- Figmaから動画化

## 5. 発信

- XおよびNoteで何件か発信を行い様子見
- ダメでもそれが1作品となりオーディエンスの期待につながる

## 5. 継続可否検討

- 反応があれば継続
- なければ次のアイデアへ
