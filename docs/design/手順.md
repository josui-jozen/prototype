# フロントエンド開発ワークフロー

- Figmaでデザインと大まかな画面遷移の設計
- 細かな実装(レスポンシブ・アクション・ロジック)はClaudeで

## 1. 参考サイトをFigmaに取り込む

- `html.to.design`(h2d)のChrome拡張を使用して画面取り込み
- ゼロから作る場合はデザインシステムなどの情報をfigma makeへ

### 2. Figmaでデザインを調整する

- 画面やモーダル同士の遷移を定義
  - ある程度はClaudeで補完できるので、遷移は大体でOK
- 大まかなデザイン配置も設定
- レスポンシブ対応は不要
- Claudeの認識精度を上げるため、繰り返しのコンポーネントは同名に

### 3. Figma MCPからデザイン情報を取得

- まずAPIでメタデータを取り、ローカルMCPで詳細を取る
  - まず大まかに設計書を作成し、各コンポーネントごとにMCPで情報再取得
- 以下の情報を取得
  - デザインの構造・レイアウト（`get_design_context`）
  - スクリーンショット（`get_screenshot`）
  - メタデータ・ノード構造（`get_metadata`）
  - 変数定義（`get_variable_defs`）
- インタラクションは`Figma REST APIを使用`
  - ただ大抵は口頭でClaudeに伝えた方が早い

### 4. コード実装

- デザイン情報からReactコンポーネントを実装
- 画面遷移・モーダル・アニメーション・レスポンシブも実装
- データについてはAPIをモック
