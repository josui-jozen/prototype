# アーキテクチャ設計

## 概要

Next.js App Router を使用した、3層レイヤードアーキテクチャを採用する。

- **presentation**: `app/` 配下の Server Components / Client Components / Route Handlers。
- **usecase**: ビジネスロジック。Repository インターフェースにのみ依存(DIP)。副作用なし(純粋関数)。
- **infrastructure**: Repository インターフェースの実装。`app/api/` を叩いてDB・外部サービスにアクセス。

## ディレクトリ構成

```
src/
├── app/                          # Next.js App Router / presentation 層
│   ├── (routes)/                 # ページ
│   │   └── articles/
│   │       ├── page.tsx
│   │       └── actions.ts        # ページ専用 Server Actions
│   ├── _actions/                 # 複数ページで共有する Server Actions
│   ├── _components/              # 共通コンポーネント
│   ├── _hooks/                   # カスタムフック・状態管理
│   └── api/                      # Route Handlers
│
├── usecase/                      # ビジネスロジック(純粋関数)
│   ├── article/
│   │   ├── article.repository.ts # Repositoryインターフェース定義
│   │   └── article.usecase.ts    # ユースケース実装
│   ├── user/
│   │   ├── user.repository.ts
│   │   └── user.usecase.ts
│   └── ...
│
└── infrastructure/
    └── repository/
        ├── article.server.ts     # Server Component用: DBに直接アクセス(Drizzle)
        ├── article.client.ts     # Client Component用: app/apiをfetch
        ├── user.server.ts
        ├── user.client.ts
        └── ...
```

### Repository実装の使い分け

| 使用元 | 使う Repository | 実装 |
|--------|----------------|------|
| Server Components | `*.server.ts` | Drizzle でDB直接アクセス |
| Client Components | `*.client.ts` | `app/api` を fetch |

usecase のインターフェース定義は共通。server/client それぞれが同じインターフェースを実装する。

`.server.ts` は `import 'server-only'` を宣言することで、Client Component から誤って import するとビルドエラーになる。

### DI(依存性注入)の実装パターン

DI は `app/` 層が usecase を呼ぶ際に repository を引数として渡すことで行う。

**Server Component(ページ読み込み)**

```ts
// app/articles/[id]/page.tsx
import { getArticle } from '@/usecase/article/article.usecase'
import { articleServerRepo } from '@/infrastructure/repository/article.server'

const article = await getArticle(articleServerRepo, params.id)
```

**Server Action(Client Componentからの更新)**

```ts
// app/articles/actions.ts
'use server'
import { publishArticle } from '@/usecase/article/article.usecase'
import { articleServerRepo } from '@/infrastructure/repository/article.server'

export const publishArticleAction = (draft: Draft) =>
  publishArticle(articleServerRepo, draft)
```

```ts
// app/articles/editor.tsx (Client Component)
'use client'
import { publishArticleAction } from './actions'

// Client Component は Server Action を呼ぶだけ。repository を知らない
<button onClick={() => publishArticleAction(draft)}>公開</button>
```

`.client.ts` が必要になるのは SWR / React Query でブラウザから定期的に fetch するケースのみ。ほぼ全ての操作は Server Components / Server Actions で完結するため、`.server.ts` を使う。

### app/api(Route Handlers)

- Client Components からのデータ取得
- 外部向けAPI(API-01: 記事操作API)
- 認証チェック(NextAuth セッション検証)

## 技術構成

| レイヤー | 技術 |
|----------|------|
| フレームワーク | Next.js(App Router) |
| ホスティング | Vercel |
| DB | Neon(PostgreSQL)+ Drizzle ORM |
| 画像ストレージ | Cloudflare R2 |
| エディタ | CodeMirror 6 |
| 認証 | NextAuth.js(Google OAuth) |
