# 要件定義

## 概要

※これはユーザーニーズ理解のための、コピープロトタイプ実装です。

SNSのような賑やかな広場ではなく、個人的な部屋のような空間を提供する。日記やエッセイなど、有益性より個人的な記録・表現を重視する。

- **必要最小限の機能** — コミュニティ機能を意図的に排除し、書くことに集中できる環境を提供する
- **静かなつながり** — フォロワー数やいいね数を可視化せず、控えめな手段でのみ他者とつながる

## 機能要件

### 認証

- Googleログイン

### 記事執筆

- リッチテキスト編集
  - マークダウンベースのリッチテキストエディタ
  - リンクカード埋め込み(一般サイト・Youtube・X)
  - ボタンによる画像挿入
- エディタのムード変更
- 手動保存
- 文字数カウンター

### 記事管理

- ゴミ箱（記事削除と復元）
  - 削除は記事カードのポップオーバメニューから選択
- 記事ごとのタグ付け
- 記事ごとのバージョン管理
- 公開
  - 公開範囲設定が設定可能(自分だけの未公開・他人も見れる公開)
  - 記事のプレビューURLでそのままアクセス(READのアクセス権限だけを変える感じ)

### ユーザー管理

- プロフィール（名前だけでOK）
- 記事の見た目
- mdエクスポート

### ダッシュボード

- 読まれた数とユーザーの滞在時間

### 感想レター

- ログイン不要で自由記入メッセージ・スタンプを筆者に送信
- レター内でスタンプを押せる

### 購読

- 購読(フォロー)した筆者の更新一覧を表示

### コンセプト違反のため未実装とする機能

- いいねボタン
- コメント欄
- フォロワー数の表示
- リポスト / シェア数の表示
